# 04 — المحاضر واقتراح الأشخاص والأسلحة

## 1. الغرض

عند كتابة محضر حادث، يُدخل المستخدم **أسلوب الجريمة + المكان + الزمان** فيقترح النظام تلقائياً:

- أشخاص (من قاعدة المسجّلين) مرتبطين بأنماط مشابهة
- أسلحة ظهرت في حوادث مشابهة

## 3. القوائم المرجعية

### أنواع الجريمة `crime_types`

```
سرقة، قتل، خطف، احتيال، مخدرات، تحرش، بلطجة،
تخريب، تهريب، غسيل أموال، إرهاب، أخرى
```

### أساليب الجريمة `crime_methods`

```
مسلح، بالإكراه، كسر باب، احتيال هاتفي،
منزلية، شارع، سيارة، تعاطف، إلكتروني،
جماعي، فردي، ليلي، نهاري، أخرى
```

### الأسلحة `weapons`

| الحقل | النوع | ملاحظات |

## 4. محرك الاقتراح

### 4.1 المدخلات

```json
{
  "crime_type_id": 2,
  "crime_method_id": 5,
  "occurred_at": "2026-03-14 23:00",
  "governorate": "القاهرة",
  "district": "المعادي"
}
```

### 4.2 معادلة التقييم

| العامل | الوزن | طريقة الحساب |
|--------|-------|-------------|
| نفس أسلوب الجريمة | 35% | تطابق `crime_method_id` |
| قرب المكان | 30% | نفس الحي = 100%، نفس المحافظة = 60% |
| قرب الزمان | 15% | نفس الفترة (ليل/صباح/مساء) + نفس يوم الأسبوع |
| نفس نوع الجريمة | 10% | تطابق `crime_type_id` |
| تكرار الشخص | 10% | عدد حوادث سابقة بنمط مشابه |

### 4.3 وزن نوع المسجّل في الاقتراح

| النوع | مضاعف النتيجة |
|-------|--------------|
| مسجّل A | × 1.0 |
| مسجّل B | × 0.8 |
| زائر | × 0.5 |

### 4.4 عتبات العرض

| النتيجة | التصنيف |
|---------|---------|
| ≥ 70% | اقتراح قوي — يظهر أولاً |
| 40–69% | اقتراح محتمل |
| < 40% | لا يُعرض |

### 4.5 اقتراح الأسلحة

من `report_weapons` للمحاضر المطابقة:
- نفس `crime_method` → أسلحة مستخدمة فيها
- ترتيب حسب تكرار الظهور

## 6. واجهة المحضر

- تحديث الاقتراحات **حياً** (debounce 500ms) عند تغيير أي حقل
- تمييز بصري لنوع المسجّل (A أحمر / B برتقالي / زائر أصفر)
- إمكانية إضافة شخص جديد كـ **زائر** مباشرة من المحضر
- checkbox لقبول/رفض كل اقتراح
- عند القبول: `link_source = suggested` + `confidence_score`
